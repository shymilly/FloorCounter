Алгоритм 1: Проекция горизонтальных границ (Аделя)
Суть алгоритма
Алгоритм основан на анализе вертикальной проекции яркости изображения. Ищутся резкие перепады яркости между небом, зданием и землёй, что позволяет определить границы здания, а затем — этажи по горизонтальным полосам.

Этапы работы
Этап	                          Описание
1. Конвертация в оттенки серого	Преобразование RGB-изображения в градации серого
2. Вертикальная проекция       	Суммирование яркости по каждому столбцу изображения
3. Поиск границ здания	        Обнаружение резких перепадов (небо→здание, здание→земля)
4. Горизонтальная проекция	    Анализ яркости по строкам внутри здания
5. Детекция этажей	            Поиск повторяющихся горизонтальных паттернов

Сильные стороны
Быстрый (среднее время ~15-25 мс на изображение)

Не требует настройки параметров

Хорошо работает на контрастных изображениях

Слабые стороны
Чувствителен к освещению (тени, засветы)

Плохо работает на сложных фасадах с балконами и декором

Требует чёткой границы между зданием и небом

Рекомендуемые параметры
Параметр    	              Значение
Порог градиента         	  30-40
Минимальная высота здания	  100 px
Размер ядра сглаживания	    5x5

Алгоритм 2: Автокорреляция вертикального профиля (Алина)
Суть алгоритма
Алгоритм анализирует вертикальный профиль яркости (усреднённую яркость по горизонтали) и ищет в нём повторяющиеся паттерны с помощью автокорреляции. Расстояние между пиками автокорреляции соответствует высоте одного этажа.

Этапы работы
Этап	                                Описание
1. Выделение вертикальной полосы	    Берётся центральная полоса изображения (20-30% ширины)
2. Вертикальный профиль	              Усреднение яркости по горизонтали → 1D сигнал
3. Нормализация	                      Вычитание среднего, приведение к нулевому среднему
4. Автокорреляция	                    Вычисление автокорреляционной функции сигнала
5. Поиск пиков	                      Определение доминантного периода (высоты этажа)
6. Расчёт этажей	                    Этажи = Высота здания / Период

Сильные стороны
Устойчив к шуму и мелким деталям

Не требует чётких горизонтальных линий

Хорошо работает на современных стеклянных фасадах

Слабые стороны
Требует настройки диапазона периодов (minPeriod, maxPeriod)

Может ошибаться на зданиях с неравномерной высотой этажей

Чувствителен к перспективным искажениям

Рекомендуемые параметры
Параметр	      Значение
minPeriod      	25-35 px
maxPeriod	      80-100 px

Полоса профиля	20-30% ширины
Порог пика	0.4-0.6

Алгоритм 3: Поиск горизонтальных линий Хаффа (Катя)
Суть алгоритма
Алгоритм использует преобразование Хафа (Hough Transform) для обнаружения горизонтальных линий на фасаде здания. Каждая найденная линия соответствует границе этажа.

Этапы работы
Этап	                          Описание
1. Предобработка	              Конвертация в оттенки серого, сглаживание GaussianBlur
2. Детекция границ	            Оператор Canny для выделения контуров
3. Морфологическое усиление	    Выделение горизонтальных линий (MORPH_CLOSE)
4. Преобразование Хафа	        Поиск отрезков (HoughLinesP)
5. Фильтрация	                  Отбор только горизонтальных (угол < порога) и достаточно длинных линий
6. Кластеризация               	Группировка близких Y-координат
7. Расчёт этажей	              Этажи = Количество кластеров + 1

Сильные стороны
Интуитивно понятный результат (линии накладываются на изображение)

Хорошо работает на зданиях с чёткими горизонтальными элементами

Позволяет визуально проверить качество детекции

Слабые стороны
Чувствителен к параметрам Canny и Hough

Плохо работает на стеклянных фасадах без чётких линий

Может находить ложные линии на текстурированных поверхностях

Рекомендуемые параметры (три набора)
Параметр	                   STANDARD	  SENSITIVE	  CONSERVATIVE
angleThreshold	              10°	         12°	         8°
minLengthRatio	              0.20	      0.15	        0.30
cannyThreshold1/2           	50/150	   30/100       	70/200
houghThreshold               	80	         60          	100
minLineLength	                50 px	      40 px       	70 px

Выводы по алгоритмам
Проекция горизонтальных границ — самый быстрый, но наименее точный. Хорош для предварительной оценки.

Автокорреляция вертикального профиля — наиболее универсальный. Работает даже при отсутствии чётких линий. Требует точной настройки диапазона периодов.

Поиск горизонтальных линий Хаффа — даёт наилучшую визуализацию. Оптимален для зданий с выраженной этажной структурой. Требует подбора параметров под тип здания.
